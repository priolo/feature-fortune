The Workflow

PROPOSE

Define a feature for any GitHub repository.

ASSIGN

Assign it to yourself or another developer.

RELEASE

Declare the improvements implemented.

ACCEPT

Review the work and accept the release.

PAY

Direct Stripe payout to the developer.

How it Works

CREATE

An author links to a public GitHub repository and describes the functionality he or she wants implemented.

ACCEPT

The implementation will be accepted by a developer, preferably the repository owner.

FUND

Like a project? Contribute a feature reward. Payment will only be made after the feature has been declared complete.

DEV

The developer will gladly dedicate his time to improving his repo if he receives money.

CHECK

When the developer releases the feature, the author checks it and, if necessary, declares it completed.

PAY

After a few hours the developer receives all the accumulated donations

We Built PUCE because
We Love Open Source.

Open source sustainability is a hard problem. Many developers maintain critical infrastructure for free while companies benefit. PUCE is an experiment in direct feature-level funding.

We believe that by allowing users to directly fund specific improvements, we create a more responsive and healthy ecosystem for everyone.

NO FEES. PERIOD.

The platform receives no compensation. Every cent (minus Stripe's transaction fee) goes directly to the developer who solves the task. This is our contribution to the community.
